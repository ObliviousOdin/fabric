# Fabric

Fabric is a local-first agent engine: one profile-scoped core across the CLI,
native desktop app, web dashboard, messaging gateway, API, and IDE integrations.
It supports personal subscriptions/OAuth, API providers, native local Ollama,
durable memory, skills, scheduled work, and delegated agents.

## Install from the Fabric repository

On macOS, Linux, or WSL2:

```bash
curl -fsSL https://raw.githubusercontent.com/ObliviousOdin/fabric/main/scripts/install.sh | bash
fabric version
```

Then configure one model route and prove it works:

```bash
fabric model
fabric status --deep
fabric chat -q "Summarize the README in this workspace."
```

Launch the source-built native desktop app with:

```bash
fabric desktop
```

See [Install Fabric](/getting-started/installation) for platform and
distribution details.

## One core, multiple surfaces

- **CLI and TUI** — direct terminal work, scripting, and approvals.
- **Desktop** — chat, projects, previews, providers, local models, memory,
  skills, messaging, and operations in a native window.
- **Web dashboard** — loopback-first administration and a remote backend when a
  real authentication boundary is configured.
- **Gateway** — long-running conversations across supported messaging channels.
- **API/ACP/MCP** — application, editor, and capability integrations.

These surfaces share the selected Fabric profile. Provider credentials, memory,
skills, and sessions do not need to be re-entered for each UI.

## Safety defaults

Fabric keeps behavioral settings in `~/.fabric/config.yaml`, secrets in the
profile's private auth store or `.env`, and dangerous commands behind approvals.
Local model inference is not automatically an air-gap claim: `local_ai` limits
participating AI/memory routes, while a whole-process `air_gapped` mode remains
fail-closed until its network boundary is verified.
