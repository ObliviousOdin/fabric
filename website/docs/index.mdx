## The shortest useful path

On macOS, Linux, or WSL2, install Fabric and prove one model route before adding
more providers or automation:

```bash
curl -fsSL https://raw.githubusercontent.com/ObliviousOdin/fabric/main/scripts/install.sh | bash
fabric model
fabric status --deep
```

Then run `fabric` and ask it to complete a small, observable task in a non-secret
workspace. A model appearing in a picker is not the same as a successful request;
the [Quickstart](/getting-started/quickstart) shows the complete verification loop.

## Profiles and the web experience

Your selected profile owns an agent's model routes, credentials, memory,
skills, approval policy, and sessions across supported interfaces. In the web
experience, **Workspace** organizes agent work and **Admin** exposes runtime
controls. A profile remains an isolated agent configuration; it is not a
tenant, team workspace, site, or human role.

<div className="docs-hub-strip">

- **See and compose work:** [Desktop](/user-guide/desktop) or [Fabric Workspace](/user-guide/workspace-admin)
- **Stay in the terminal:** [CLI](/user-guide/cli) or [TUI](/user-guide/tui)
- **Keep a conversation running:** [Messaging gateway](/user-guide/messaging/)
- **Connect applications and editors:** [API server](/user-guide/features/api-server), [ACP](/user-guide/features/acp), or [MCP](/user-guide/features/mcp)

</div>

:::info Current web delivery
The responsive Workspace/Admin shell, route structure, Home, and three-panel
Chat are available now. Server-enforced enterprise tenancy and roles, a durable
Work Board service, durable approvals, unified activity, and the typed Memory
ledger remain staged backend contracts. The
[delivery map](/user-guide/workspace-admin#what-is-available-now) keeps this
boundary explicit.
:::
